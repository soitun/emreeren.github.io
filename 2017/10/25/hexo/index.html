Hexo

Published on Wednesday, October 25th 2017 at 4:42
Last updated on Wednesday, October 25th 2017 at 5:20

Bu siteyi Hexo kullanarak yapıyorum. Hexo bir Static Site Generator. Yani bir Wordpress gibi sunucuda çalışan bir yazılım değil. Hexo bilgisayarımızda çalışıyor ve yazılarımızı yazdığımız Markdown dosyalarını kullanarak HTML, CSS dosyalarını üretiyor. Kendi bilgisayarımızda sitemizin son görünümünü kontrol ettikten sonra oluşan dosyaları upload ediyoruz.

Wordpress tabii hiç tartışmasız bir web sitesi yapmak için kullanılabilecek en iyi seçeneklerden bir tanesi. Ancak sıkıntıları da yok değil. Örneğin sunucumuzda PHP kodlarının çalışabilmesi, bir veritabanı olması gibi gereksinimler var. Sunucumuzda wordpress çalıştırdığımız zaman işletim sisteminin, sunucu yazılımlarının, wordpress’in kendisinin ve kullanılan bütün eklentilerin periyodik olarak güncellenmesi gerek. Aksi takdirde wordpress için satın aldığınız temanızın kullandığı eklentilerden birisinin bir açığı nedeniyle güvenlik sorunları oluşabiliyor.

Bu problemler nedeniyle Wordpress’in sunduğu güzel özelliklere ihtiyaç olmayan küçük web siteleri için Static Site Generator yazılımları kullanmak iyi bir alternatif. Sadece statik dosyalar üretildiği ve sunucu tarafında hiç bir kod çalışmadığı için güvenlik problemleri en aza iniyor. Tabii sunucu yazılımlarına ihtiyacınız olmadığı için benim yaptığım gibi Github Pages gibi ücretsiz hosting çözümlerini de kullanabiliyorsunuz. RAM veya CPU hızı konuları da artık o kadar önemseyeceğimiz bir konu değil.

Hemen her platformda çalışan bir Static Site Generator yazılımı var. Ben Javascript tabanlı olduğu ve zaten bilgisayarımda Node.js yüklü olduğu için Hexo tercih ettim ama bu konuda en popüler proje Jekyll. Github Pages da buna direk olarak destek verdiği için çok basitçe kullanıma geçebiliyorsunuz.

Tabii yazılarımızı web üzerinden janjanlı editörlerimizi kullanarak yazma şansımız yok. Yazılarımızı bütün SSG projelerinin desteklediği Markdown biçimini kullanarak yazıyoruz. Bu tabii herkes için avantaj değil ancak yazılımcılar genellikle işlerini Notepad gibi Text Editor tarzı çalışan araçlar ile yaptıkları için hoşlanabilirler. İşini daha çok CLI araçları ile yapan yazılımcılar command line üzerinde vim ile iki satır yazıp git ile publish yapabilmeyi pratik bulacaklardır.

Yazıları git ile publish etmek eğer web sitesini güncelleyecek ekibin git kullanma bilgisi varsa beraber çalışabilme açısından şahane bir imkan. Henüz daha yeni başladığım için fazla farkında değilim ama sanırım en büyük problem yazıların altına okuyucuların yorum ekleyememesi olacak. Belki Disqus gibi çözümler kullanarak pratik bir şekilde halledilebilir.

Bulduğum hazır bir temayı yükledim. Ufak tefek değişiklikleri artık zamanla yaparım. Şimdilik bu kadar. Biraz Hexo’nun detaylı özelliklerine bakayım ilginç şeyler bulursam yazarım.
